Git Workflow Cheat Sheet Windows + Ubuntu VM

Fast commands for working across two machines (push from one, pull on the other).

Daily Workflow (Two Machines)

# Before you start (on any machine)
git pull

# After you finish edits
git add .
git commit -m "Describe what you changed"
git push

Tip: Always git pull before editing to avoid conflicts.

Commit & Push a Single File

git add path/to/file.ext
git commit -m "Update file.ext"
git push

First-Time Setup (Per Machine)

git config --global user.name  "FerasMorsi"
git config --global user.email "feras-m@hotmail.com"

Connect Local Folder to GitHub (One-Time)

git remote add origin https://github.com/FerasMorsi/REPO_NAME.git
git branch -M main
git push -u origin main

Clone on Another Laptop

git clone https://github.com/FerasMorsi/REPO_NAME.git
cd REPO_NAME
git pull

Status & History

git status
git log --oneline --graph --decorate --all

Branch Basics (Optional)

git switch -c feature/my-change     # create & switch
git switch main                     # go back to main
git merge feature/my-change         # merge into current branch
git push -u origin feature/my-change

Undo / Restore (Safe)

git restore path/to/file.ext        # discard local file changes
git reset                           # unstage (keep changes)

Pull Problems (Quick Fixes)

git pull --rebase
# If you really want local to win (destructive):
git push -u origin main -f

SSH (No Password Prompts)

ssh-keygen -t ed25519 -C "feras-m@hotmail.com"
# Press Enter for defaults
cat ~/.ssh/id_ed25519.pub   # copy to GitHub ‚Üí Settings ‚Üí SSH and GPG keys
# After adding the key, switch your remote to SSH:
git remote set-url origin git@github.com:FerasMorsi/REPO_NAME.git
